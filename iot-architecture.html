System Architecture

Final GitHub Pages architecture for mathematics learning analytics

The final implementation is intentionally simple and static so it can run directly on GitHub Pages. Instead of depending on backend deployment, the project uses seeded LMS-like event data, client-side analytics, rule-based feedback logic, and session-backed runtime updates inside a React 18 application.

End-to-End Flow

Final system flow

1. Seeded academic records

Students, LMS connectors, and mathematics learning events are preloaded for demonstration.

2. Browser analytics layer

Client-side logic computes mastery, trends, class averages, and risk bands.

3. Feedback generation

Rule-based feedback is created for students and teacher intervention planning.

4. Role-based dashboards

Student, Teacher, and Admin pages present insights and runtime controls.

Core Frontend Modules

Main implemented layers

  • Static data layer
  • Analytics computation layer
  • Feedback generation layer
  • Role-based React pages
  • Session-backed runtime state

Technology Stack

Final technologies used

  • React 18 for interface structure
  • D3.js for trend and mastery charts
  • JavaScript for analytics, feedback, and runtime state
  • sessionStorage for temporary runtime persistence
  • GitHub Pages for public hosting

Why This Structure

Reason for choosing a static final architecture

The final structure was simplified so the deployed project exactly matches the hosted result. This avoids the mismatch between a static host and an undeployed backend. The submitted version is therefore easier to run, easier to explain, and easier to defend during project evaluation.
