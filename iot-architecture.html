System Architecture

Approved microservices pipeline for mathematics learning analytics

The architecture now centers on Python Flask microservices, Apache Kafka messaging, Apache Cassandra storage, and a React 18 plus D3.js visualization frontend. The old monolithic browser-only demo has been removed from the core implementation path.

End-to-End Flow

Service pipeline

1. LMS and platform inputs

Moodle, Canvas, and online mathematics systems produce learning events.

2. Data Collector Service

Ingests events and prepares them for streaming through Kafka topics.

3. Processor and Feedback Services

Compute mastery, learner risk, topic weakness, and role-specific recommendations.

4. Visualization and LMS Integration

Expose live dashboards and synchronized academic insights to stakeholders.

Core Services

Backend modules

  • Data Collector Service
  • Data Processor Service
  • Feedback Generator Service
  • Visualization Service
  • Integration Service

Platform Stack

Approved technologies

  • Python Flask for each microservice
  • Apache Kafka for message-driven event flow
  • Apache Cassandra for scalable persistence
  • React 18 and D3.js for real-time analytics views
  • Jupyter Notebook for exploratory analysis and experimentation

Why This Structure

Reason for removing the old root prototype

The old root application used plain HTML, CSS, and JavaScript as a single-page prototype. That was useful for early demonstration, but it did not match the approved project language and architecture. The project now uses a structure that directly reflects the approved stack and problem statement.
