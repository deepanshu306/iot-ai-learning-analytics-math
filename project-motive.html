Final Year Project Topic

IoT-Enabled AI-Driven Learning Analytics and Personalized Feedback System for Mathematics Education

This project is designed to combine Internet of Things, learning analytics, and intelligent feedback into a smart mathematics education system. Instead of depending only on manual observation and final marks, the system uses software-generated data from simulated IoT classroom nodes and digital assessment activity to identify weak concepts, track learning progress, and support data-driven academic intervention.

Core idea

Why this project matters

Traditional assessment usually stops at score reporting. This project extends that process with simulated connected data collection, diagnosis, interpretation, and intervention so that the result becomes useful for both learners and teachers.

Problem Statement

Existing issue

In many classrooms, mathematics evaluation is limited to marks, percentages, and final remarks, while important learning signals remain untracked in real time. Such evaluation does not clearly explain why a student is underperforming, which concept is weak, whether mistakes are repeated, how classroom participation changes, or how performance evolves across attempts. Teachers also face difficulty in manually identifying patterns across many learners, especially when quick intervention is needed.

Project Motive

Main motive

The main motive of this project is to develop a smart academic support system that combines IoT-based classroom data acquisition with learning analytics and personalized feedback generation. In the current prototype, the IoT layer is represented through software-simulated nodes rather than physical devices. These virtual nodes generate learner-related inputs such as attendance events, quiz activity, and engagement logs, which are then analyzed together with mathematics assessment results to detect topic-level learning gaps, repeated misconceptions, and performance trends. In this way, the project moves from simple digital testing to IoT-assisted, data-driven educational support without depending on hardware.

IoT Relevance

Why it fits an IoT-based project

From an IoT perspective, this project is suitable because it is based on collecting, transmitting, processing, and visualizing educational data generated through simulated connected endpoints. The project can be positioned as a smart classroom system in which virtual IoT nodes capture attendance, quiz participation, engagement-related signals, and assessment activity, while the software layer performs analytics and feedback generation. From a CSE perspective, it remains strong because it combines IoT architecture concepts, real-time style data handling, dashboards, decision logic, and intelligent recommendation workflow without requiring hardware procurement.

Prototype Strategy

How it stays IoT-based without hardware

The current version uses a hardware-free IoT model. Instead of real sensors or boards, the system simulates IoT behavior through virtual data sources, event generation, and a centralized monitoring dashboard. This preserves the essential IoT pipeline of data generation, transmission logic, processing, analysis, and monitoring, while keeping the project practical for software-only implementation.

IoT Integration

How IoT fits into the system

Attendance node simulator

Software-generated attendance events act as virtual IoT input for learner presence tracking.

Quiz event node

Quiz attempts, response timing, and completion events behave as connected learning-node data.

Engagement event logger

Interaction logs and simulated participation signals model classroom engagement without sensors.

Central analytics dashboard

The web application acts as the monitoring, analytics, and feedback layer for the virtual IoT classroom setup.

Objectives

What the project aims to do

Collect classroom data

Capture attendance, quiz activity, and learner interaction data through simulated IoT endpoints.

Analyze performance

Track score, accuracy, time taken, topic mastery, and performance trends from mathematics quiz attempts.

Detect weak areas

Identify weak chapters such as algebra, trigonometry, and calculus using response-level analysis.

Generate feedback

Produce personalized recommendations based on low accuracy, slow pace, repeated mistakes, and collected learning signals.

Support teachers

Help teachers monitor class trends, compare students, and choose suitable intervention strategies.

Build a hardware-free IoT prototype

Create a practical smart classroom prototype that demonstrates IoT workflow through virtual nodes and cloud-style analytics.

Expected Outcome

Result of the system

The expected outcome is an IoT-enabled mathematics learning support system that helps students and teachers move from passive score viewing to active performance monitoring. It should provide a clear analytical view of attendance-linked learning behavior, subject weakness, error patterns, and intervention needs through a central dashboard, even when the IoT layer is represented through software simulation.

Future Scope

Possible extension

Future versions can include MQTT-style messaging, cloud-connected IoT services, backend storage, adaptive question selection, predictive performance modeling, ML-based recommendation engines, AI-supported tutoring suggestions, and optional physical node integration if hardware is added later.
