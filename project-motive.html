Project Motive

Why this project matters

Traditional mathematics assessment often ends with marks and rank. This project extends that process into diagnosis, interpretation, and intervention. The motive is to turn raw learning records into understandable analytics and personalized feedback that can be used by students, teachers, and academic coordinators for better learning outcomes.

Problem Description

What needs to be solved

Online mathematics practice systems and LMS platforms generate large amounts of learner activity data, but students and teachers often receive only final marks instead of meaningful interpretation. As a result, weak topics, repeated mistakes, and progress trends are not clearly identified in time for improvement.

Objective

What the final project delivers

The project delivers a role-based analytics platform for mathematics education that brings together personalized student feedback, teacher-side class analytics, and academic administration in one integrated web application. It converts mathematics learning records into meaningful visual insight and guided learning support.

Implemented Modules

Final submission structure

Student Dashboard

Shows topic mastery, performance trend, recent activity, and personalized feedback.

Teacher Dashboard

Shows class average, risk distribution, topic heatmap, and selected-student drill-down.

Admin Dashboard

Supports runtime user creation, LMS connector updates, and simulated event ingestion.

Analytics Engine

Computes topic mastery, score averages, completion rate, and learner risk from seeded events.

Feedback Engine

Generates rule-based recommendations for students and teacher intervention notes.

GitHub Pages Deployment

Publishes the final static React application directly from the repository.

Why It Fits IoT Based

Software-only connected learning flow

The project is positioned as IoT-based through its connected academic event flow. LMS updates, learner attempts, connector status, analytics processing, and dashboard monitoring operate as linked educational events within one integrated platform.

  • Mathematics learning platforms act as connected educational endpoints.
  • Seeded LMS-style records simulate continuous academic event generation.
  • The analytics layer centralizes interpretation and feedback generation.
  • The GitHub-hosted dashboard acts as the monitoring and visualization layer.

Project Highlights

Key outcomes of the final system

  • Student page for personalized feedback and topic-wise performance review.
  • Teacher page for class analytics and learner drill-down.
  • Admin page for runtime management and event simulation.
  • Interactive mathematics dashboards with visual charts and performance indicators.
  • Connected LMS-style academic event flow represented in one public web platform.
  • Publicly accessible deployment that can be demonstrated directly during evaluation.
