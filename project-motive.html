Project Motive

Why this project exists in the approved stack

The project is meant to solve a real academic problem: mathematics learning platforms generate large amounts of student performance data, but that data rarely becomes useful feedback for students, instructors, and administrators in time to improve outcomes. The motive is to convert disconnected activity logs into structured learning analytics and personalized intervention pathways.

Problem Description

What needs to be solved

Online mathematics platforms, LMS systems, and digital assessments continuously produce learner activity data. In most institutions, this data remains fragmented across systems, so neither teachers nor academic administrators get a clear view of concept mastery, repeated mistakes, weak topics, or student risk in real time.

Approved Objective

What the system must achieve

The system must provide real-time learning analytics and personalized feedback through a scalable and secure microservices architecture. In this implementation it supports role-based feedback for students, instructors, and administrators, integrates with existing LMS platforms, and exposes live visual insights through a modern frontend.

Proposed Solution

How the architecture answers the objective

Data Collector Service

Accepts mathematics learning events from online platforms and LMS connectors.

Data Processor Service

Transforms raw events into topic mastery, trend data, and learner risk summaries.

Feedback Generator Service

Builds tailored feedback for students and instructors from the analyzed data.

Visualization Service

Delivers real-time dashboards using React 18 and D3.js.

Integration Service

Coordinates Moodle, Canvas, and similar LMS integration flows.

Kafka + Cassandra

Support event-driven messaging and scalable storage for learner activity streams.

Why It Fits IoT Based

Connected event flow without depending on hardware boards

The project qualifies under the IoT-based problem type because it treats LMS and mathematics platform activity as connected event streams moving through a distributed system. The key IoT idea here is not a physical sensor board, but connected data acquisition, message flow, processing, and live monitoring across services.

  • Multiple platforms act as connected educational endpoints.
  • Kafka models distributed event transmission between services.
  • Cassandra stores large-scale event history for analytics.
  • React and D3 render a monitoring and feedback dashboard in near real time.

Implemented Roles

What is included in this submission

  • Student page for personalized feedback, topic mastery, and performance trend.
  • Teacher page for class analytics, student drill-down, and LMS connector monitoring.
  • Admin page for runtime user management, connector control, and event ingestion.
  • Parent-facing page is intentionally excluded from the current submission scope.
